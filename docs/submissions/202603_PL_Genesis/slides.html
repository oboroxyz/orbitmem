PL_Genesis: Frontiers of Collaboration Hackathon

Problem

IPFS gives you content-addressable storage — but not a database.
Three things are missing:

🔓

No Encryption

IPFS stores data in the open. No built-in per-record encryption or fine-grained access control.

🚫

No Authentication

No identity layer, no signed requests. Any node can access any content if it has the CID.

🔍

No Discovery

No search, no quality signal. You need a CID to find anything — decentralized data is invisible.

OrbitMem

A decentralized data layer for the agentic web — encrypted vaults, on-chain discovery, and verifiable data trust, designed for both humans and AI agents.

Key Features

🌐

Fully Decentralized

Built on IPFS and OrbitDB — no central servers, no single point of failure. Offline-first, censorship resistant.

🔒

Encrypted Vaults

P2P data vaults with per-path visibility control. End-to-end encryption — only authorized parties can decrypt.

✅

Discovery & Trust

On-chain data discovery and quality scoring via ERC-8004. Every data entry is rated by humans and agents.

✍️

Signed Transport

ERC-8128 signed HTTP with Passkey, ETH or Solana Wallet. Every request is cryptographically verified across chains.

Demo 2: Agent Research & Data Trust

AI agents publish, discover, and build trust around data — autonomously.

# Agent A — publish & register
$ npx orbitmem vault store \
research/q1 '{"topic":"AI"}'
$ npx orbitmem register
# Agent B — discover & rate
$ npx orbitmem discover \
--schema research --min-score 70
// Rate via SDK
await client.rateData(tokenId, {
score: 90, tag: 'accurate'
});

Key Features

  • ▸Register data on-chain — mint ERC-721 receipt
  • ▸Discover by schema, tags, quality score
  • ▸Rate data — per-tag reputation scoring
  • ▸Reputation-gated access via Lit Protocol
  • ▸Virtuous cycle — quality ↑ visibility ↑
